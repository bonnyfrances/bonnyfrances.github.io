Remote Connection to Hosted Database

Introduction

With pgAdmin4 installed, we can now establish a connection between the two in order to build the structure and populate the database with content.

Check the Access Control Source

pgAdmin Remote Connection

  1. Open the pgAdmin application.
  2. On the left side, should be a "Servers" panel. This is where we will build the connection.
    1. Right-click on the Servers name.
    2. Select Register > Server. A dialog box should open:
    3. Provide a descriptive name (e.g. CSE340 Database). Leave all other settings on this screen alone.
    4. Click "Connection" in the top bar of the dialog box.
    5. Using the file with your database connection info linked previously for all following credentials,
    6. Copy the IP address
    7. Return to pgAdmin and paste the IP address into the "Host name / address" dialog box.
    8. The value of the Port should be "5432". That is correct, so don't change it.
    9. In the pgAdmin dialog box, paste the database name into the "Maintenance database" text box.
    10. In the pgAdmin dialog box, paste the Username into the "Username" text box.
    11. In the pgAdmin dialog box, paste the Password into the "Password" text box and toggle "Save Password" option.
    12. Leave the other settings on this screen alone.
    13. Click "Parameters" in the top bar of the dialog box.
    14. Change, if needed, the "SSL Mode" to "Require".
    15. Click the "Save" button.

Close the Connection

On the Free tier that offers, it is important to not overuse time or resources. It is recommended that when you are done with your interaction between pgAdmin and the database server that the connection be closed. To do so:

  1. Right-click on the Render server object in the "Servers" panel of pgAdmin.
  2. Select "Disconnect from Server".
  3. You're done.

Conclusion

As you can see, establishing this connection requires a fair amount of work. You should only have to repeat this when your Render.com database expires and a new one is built.
